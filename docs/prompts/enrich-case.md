# プロンプト: 既存事例を追加文献で補完する

既存の事例で「調査中」になっている項目を、追加の文献情報で補完するためのプロンプトです。

---

## 使い方

1. 下記のプロンプトをコピーする
2. 補完したい事例の `case.json` の内容を貼り付ける
3. 追加の文献URL・PDFを添付する
4. AIに送信する
5. 出力された更新版JSONで `case.json` を上書きする

---

## プロンプト本文

````
以下の既存事例JSONと追加文献を照合し、「調査中」の項目を補完してください。

## 既存の事例JSON
```json
【ここに既存の case.json の内容を貼り付け】
```

## 追加文献
【ここに追加文献のURL、またはPDFを添付】

## 作業ルール

1. **更新対象**: 「調査中」と記載されている項目のうち、追加文献から情報が得られるものを更新する
2. **既存内容の保持**: 既に記入済みの項目は変更しない。ただし、より正確・詳細な情報が文献にある場合は改善してよい
3. **sources の追加**: 追加文献を sources 配列に追記する（既存のsourceは削除しない）
4. **updated_at**: 本日の日付に更新する
5. **editorial_notes**: 補完できなかった「調査中」項目があれば、何が不明なのかをメモとして残す
6. **推測しない**: 文献に明記されていない情報は「調査中」のまま残す

## data_flow 図の改善

既存の図のノードについても、追加文献の情報で label をより具体的にできる場合は更新してください。
例:
- 「合成データを生成」→「CTGANで合成データを生成」（手法名が判明した場合）
- 「成果を達成」→「検出精度95%を達成」（数値が判明した場合）

## 出力形式

更新後の完全なJSONを出力してください。
JSONの後に、変更点のサマリーを箇条書きで示してください：
- 更新した項目と内容
- まだ「調査中」のまま残っている項目
- 新たに判明した情報の出典箇所
````
