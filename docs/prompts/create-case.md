# プロンプト: 文献から事例JSONを作成する

以下のプロンプトをAI（Claude等）に入力し、文献のURLやPDFを添付することで、`case.json` のドラフトを生成できます。

---

## 使い方

1. 下記のプロンプトをコピーする
2. `【ここに文献のURL、またはPDFを添付】` の部分を差し替える
3. 必要に応じて `【ここに事例ID】` を指定する（例: `seed-0013`）
4. AIに送信する
5. 出力されたJSONを `public/cases/<id>/case.json` として保存する
6. `public/cases/index.json` にIDを追加する

---

## プロンプト本文

````
以下の文献から、合成データ（Synthetic Data）の活用事例を抽出し、case.json を作成してください。

## 文献
【ここに文献のURL、またはPDFを添付】

## 出力仕様

1件の事例につき1つのJSONを出力してください。文献中に複数の事例が含まれる場合は、事例ごとに分けて出力してください。

各JSONは以下のスキーマに従ってください：

```json
{
  "id": "【ここに事例ID（例: seed-0013）】",
  "title": "（後述のガイドラインに従って作成）",
  "region": "国内 or 国外",
  "domain": "金融 / 医療 / 公共 / 通信 のいずれか",
  "domain_sub": "分野のサブカテゴリ（任意。例: 保険, 製薬, がん臨床）",
  "organization": "実施組織名",
  "usecase_category": ["下記の選択肢から1つ以上"],
  "summary": "（後述のガイドラインに従って作成）",
  "value_proposition": "成果・効果の具体的な説明",
  "synthetic_generation_method": "合成データの生成手法（GAN, VAE, CTGAN, 統計的手法 等）",
  "safety_evaluation_method": "安全性の評価方法（k-匿名性, 差分プライバシー 等）",
  "utility_evaluation_method": "有用性の評価方法（精度比較, F値, 下流タスク評価 等）",
  "tags": ["検索用のキーワード2〜4個"],
  "editorial_notes": ["情報が不完全な箇所や要追加調査事項のメモ（任意）"],
  "sources": [
    {
      "source_type": "web or pdf",
      "title": "文献の正式タイトル",
      "url": "文献のURL",
      "note": "この文献からどの情報を参照したかの補足"
    }
  ],
  "figures": [
    {
      "type": "data_flow",
      "title": "（事例名）の合成データ活用フロー",
      "data": {
        "nodes": [
          {"id": "s1", "label": "元データの説明",       "category": "source"},
          {"id": "s2", "label": "課題・制約の説明",      "category": "constraint"},
          {"id": "p1", "label": "合成データの生成方法",   "category": "process"},
          {"id": "a1", "label": "合成データの活用先",     "category": "application"},
          {"id": "a2", "label": "得られた成果・効果",     "category": "outcome"}
        ],
        "edges": []
      },
      "note": ""
    }
  ],
  "status": "seed",
  "created_at": "（本日の日付をISO 8601形式で）",
  "updated_at": "（本日の日付をISO 8601形式で）"
}
```

## usecase_category の選択肢

文献の内容から最も適切なものを選んでください：

| カテゴリ | 判断基準 |
|---------|---------|
| 組織内データ共有 | 同一組織内の部門間でデータを共有するために合成データを使う |
| 組織間データ共有 | 異なる組織（取引先、パートナー等）にデータを提供するために合成データを使う |
| 外部分析者活用 | 外部の分析者・研究者にデータを提供して分析してもらう |
| R&D | 研究開発・学術研究の目的で合成データを生成・活用する |
| データ販売 | 合成データセットを商品として販売する |
| フィージビリティ検証 | 本格導入前の実現可能性検証・PoC用途で合成データを使う |

## 記入ガイドライン

### title
- 「〇〇における△△を目的とした合成データ活用」のように、対象領域と目的が分かる形にする
- 具体的な成果があれば盛り込む（例:「承認プロセスを6カ月から3日に短縮」）
- 30〜50文字程度を目安にする

### summary
- 「課題 → 合成データの活用 → 結果」の構造で1〜3文にまとめる
- 1文目で課題や背景、2文目で合成データをどう使ったか、3文目で成果を書く

### 情報が不明な場合
- 文献から読み取れない項目は「調査中」と記入する（空欄にしない）
- editorial_notes に「〇〇は文献に記載なし」等のメモを残す
- 無理に推測で埋めず、正確に分かる範囲だけを記入する

### data_flow 図のノード
- 各カテゴリに1〜3ノードが目安
- label は短く具体的に（15〜25文字程度）
- 数値があれば盛り込む（例:「有用性97%を達成」「6カ月→3日に短縮」）
- ノードIDは s1, s2（source列）、p1, p2（process列）、a1, a2（application/outcome列）の命名規則

### tags
- 検索で引っかかるべきキーワードを2〜4個
- domain や usecase_category と重複しないものを選ぶ
- 具体的な技術名、業務名、特徴的なキーワードなど

## 出力形式

JSONのみを出力してください（マークダウンのコードブロックで囲む）。
複数事例の場合は、事例ごとに別のコードブロックで出力し、間に簡単な説明を挟んでください。
````

---

## 補足: 複数文献から1つの事例を作る場合

複数の文献を同時に渡して「これらの情報を統合して1つの事例にまとめてください」と追記すれば、情報を統合した事例を作成できます。その場合、`sources` 配列に全ての文献を列挙してください。
